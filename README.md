# SQLazy
Say goodbye to the spaghetti-like mess of nested SQL – write queries step-by-step in natural language, and generate auditable standard SQL with 100% no hallucinations
---
## 🎯 What Pain Points Does It Solve?
Complex OLAP analytical queries written in native SQL are often filled with multi-layer nesting and ultra-long window functions. This not only leads to low development efficiency and poor readability, but more critically:
- SQL directly generated by AI often has correct syntax but incorrect semantics (hallucinations), making it unauditable and unfit for production environments;
- Debugging complex SQL is extremely difficult, with no way to locate the calculation results of each step;
- SQL dialects of different databases are incompatible, requiring repeated rewriting of a single query;
- Newcomers struggle to master advanced SQL capabilities such as window functions, multi-layer subqueries, and ordered calculations.

SQLazy completely resolves these issues through step-by-step natural language queries.
## ✨ Core Advantages
### 🔍 White-Box & Auditable, 100% Hallucination-Free
Unlike the black-box mode of pure AI-generated SQL:
- Your requirements are converted into readable, step-by-step standardized natural language, allowing you to verify the logic line by line;
- After logic confirmation, the final SQL is generated by a compiler – no semantic hallucinations whatsoever;
- Perfectly adapts to enterprise code auditing and high-reliability requirements of production environments.

### 🧩 Step-by-Step Development: Build Complex Queries Like Building Blocks
No matter how complex the analytical requirement is, it can be broken down into multiple simple steps. Each step performs just one action, with no need to worry about SQL nesting logic.

#### Typical Example: Calculate the Longest Consecutive Rising Days of a Stock
Implementing this with SQLazy is logically clear and supports step-by-step debugging:
```
t2 = stock filter where CODE equals 100046
t3 = t2 sort by DT ascending
t4 = t3 segment when CL decreases, name the segment NoRisingDays
t5 = t4 aggregate count of DT as ContinuousDays, group by NoRisingDays
Result = t5 aggregate max of ContinuousDays as max_ContinuousDays
```
The corresponding native Oracle SQL requires 4 layers of nesting + window functions, with extremely poor readability and difficulty in debugging:
```sql
WITH t3 AS (
  SELECT CODE, DT, CL FROM stock WHERE CODE = 100046
)
SELECT MAX(ContinuousDays) AS max_ContinuousDays
FROM (
  SELECT NoRisingDays, COUNT(DT) AS ContinuousDays
  FROM (
    SELECT CODE, DT, CL,
      SUM(CASE WHEN CL < col__3 THEN 1 ELSE 0 END)
      OVER (ORDER BY DT ASC) + 1 AS NoRisingDays
    FROM (
      SELECT t3.*, LAG(CL) OVER (ORDER BY DT ASC) AS col__3
      FROM t3
    ) sub__4
  ) t4
  GROUP BY NoRisingDays
) t5;
```
